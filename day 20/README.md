# React State & Events Practice Project

This repository documents my progress in learning React fundamentals through hands-on practice with state management, event handling, and controlled components.
---

## 🚀 Project Overview

This project focuses on understanding how React handles state changes, user interactions, and form inputs using functional components and the `useState` hook.

It includes multiple small components that demonstrate real-world UI behavior such as counters, form inputs, toggles, and controlled elements.

---

## 🧠 Concepts Learned

### 1. useState Hook
Used to manage dynamic data inside functional components and trigger UI updates when state changes.

---

### 2. Event Handling
React event handlers like `onClick` and `onChange` are used to respond to user actions.

---

### 3. State Updates
State is updated using setter functions, demonstrating how React re-renders UI when data changes.

---

### 4. Controlled Components
Form elements (input, textarea, select, radio buttons) are controlled using React state to keep UI and data in sync.

---

### 5. Multiple State Management
A single component can manage multiple independent state variables such as name, age, and status.

---

## 🛠️ Features

### 🔢 Counter Component
- Increments, decrements, and resets a numeric value
- Uses `useState` for state control
- Demonstrates real-time UI updates

---

### 👤 MyComponent (Multi-State Example)
- Manages multiple states:
  - Name
  - Age
  - Employment status (toggle)
- Shows how different state variables work together in one component

---

### 📝 ReactingToReact (Controlled Form)
- Input field (text)
- Number input (quantity)
- Textarea (comments)
- Dropdown (payment method)
- Radio buttons (shipping option)

Demonstrates full controlled form handling using React state.

---

## 📊 Key Takeaways

- React state drives UI updates
- User input must be controlled using state
- Event handlers connect UI actions to logic
- Complex UI can be built by combining simple state variables
- Forms in React are fully controlled components
